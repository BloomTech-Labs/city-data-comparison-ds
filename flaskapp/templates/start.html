Overview

The Citrics API‌ offers a broad range of city-related data for nearly every single city in the United States. Our primary source of data comes from the Census Social Characteristics, Economic Characteristics, Housing Characteristics, and Demographic Characteristics datasets. The following documentation will help you better understand what data is available and how to work with each endpoint.

Endpoints

City Data by ID

The City Data by ID endpoint returns all city data based on the Citrics unique ID. 

http://api.citrics/<API_KEY>/citydata/<City_ID>

Data returned:

All City Data

City Name Search (for misspelled queries)

This endpoint receives text and matches the text to similarly spelled cities. The main purpose of this endpoint is to receive a city ID’s if a user spells the city wrong. This endpoint returns at most five similarly spelled cities along with the Citrics ID which can be used for “citydata” endpoint. 

http://api.citrics/<API_KEY>/matchcity/<City_name_and_State_Name>

Data returned:

{
  "Salt Lake City, UT": {
    "ID": 17929,
    "lat": 40.7774,
    "lng": -111.93,
    "population": 195701,
    "st": "UT",
    "state": "Utah"
  }
  ....
}
